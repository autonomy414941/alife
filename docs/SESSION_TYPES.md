# Session Types

| Type | Description |
|------|-------------|
| feat | Add a new capability within the existing architecture |
| expand | Extend the system's fundamental expressiveness or representation |
| refactor | Restructure existing code without changing behavior |
| split | Extract code into focused modules to reduce file size |
| validate | Verify whether recent changes actually achieved their goal |
| benchmark | Measure performance, scaling, or resource cost |
| investigate | Diagnose why something failed or underperformed |
| strategize | Design an approach before implementing |
| spike | Exploratory prototype to test feasibility |
| test | Add or improve test coverage |
| revert | Remove ineffective or unused code/config |
| cleanup | Remove dead code, fix warnings, tidy structure |
| review | Audit a subsystem for correctness or design issues |
| visualize | Add visualization or reporting for better insight |
| synthesize | Combine findings across sessions into a coherent picture |
