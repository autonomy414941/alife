# Research Agenda

## Current Direction
Over the next month, shift from parameter-only founder-protection tuning toward a viability-aware mechanism surface that can express coexistence the current simulator cannot. Keep the evaluation loop honest by consuming the already-landed decomposition and abundance-weighted metrics, but spend the next bets on opening one concrete mechanism seam, starting with encounter topology, instead of adding more scalar sweeps.

## Why This Direction
The strongest anti-evidence is still absolute: under the strict habitat-plus-crowding null, founder grace improves delta versus static habitat by `+9` at cladogenesis threshold `1.0` but still sits at `-25.75` active clades versus null, and the refreshed March 15, 2026 canonical founder-grace / ecology-gate artifact (`docs/clade_activity_relabel_null_founder_grace_ecology_gate_horizon_2026-03-15.json`) only improves the species-conditioned active-clade delta from `-23.75` to `-20.25` at threshold `1.0` while cutting persistent activity delta from `35.49` to `3.79`. The non-species-conditioned null and species-versus-clade decomposition now exist in code and tests (`a70776b`), and abundance-weighted activity metrics landed in `662767d`, so the main blocker is no longer missing evaluation plumbing. The blocker is that the planner is still choosing among booleans and scalars inside a one-resource, dominant-only, clonal simulator while recent coexistence work argues that interacting mechanisms matter more than isolated knobs.

## Structural Constraints
`src/simulation.ts` (1989 lines), `src/activity.ts`, and `src/types.ts` still concentrate most runtime and study responsibilities. Encounter resolution has been extracted to `src/encounter.ts` with a pluggable `EncounterOperator` interface (`3eb3007`), creating the first mechanism seam, but the simulator remains locked to one fungible resource/energy currency, three mutable genome axes, greedy one-step local choice, and single-parent clonal inheritance. `TaxonHistory.timeline` and per-tick `localityFrames` remain always-on full histories, which keeps long-horizon and counterfactual work expensive. The encounter seam exists but has only one implementation, so the abstraction boundary remains untested.
