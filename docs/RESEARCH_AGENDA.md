# Research Agenda

## Current Direction
Over the next month, shift from parameter-only founder-protection tuning toward a viability-aware mechanism surface that can express coexistence the current simulator cannot. Keep the evaluation loop honest by consuming the already-landed decomposition and abundance-weighted metrics, but spend the next bets on opening one concrete mechanism seam, starting with encounter topology, instead of adding more scalar sweeps.

## Why This Direction
The strongest anti-evidence is still absolute: under the strict habitat-plus-crowding null, founder grace improves delta versus static habitat by `+9` at cladogenesis threshold `1.0` but still sits at `-25.75` active clades versus null, and the best current ecology gate only improves to `-17` while sharply reducing persistent activity on the March 14, 2026 canonical horizon panel. The non-species-conditioned null and species-versus-clade decomposition now exist in code and tests (`a70776b`), and abundance-weighted activity metrics landed in `662767d`, so the main blocker is no longer missing evaluation plumbing. The blocker is that the planner is still choosing among booleans and scalars inside a one-resource, dominant-only, clonal simulator while recent coexistence work argues that interacting mechanisms matter more than isolated knobs.

## Structural Constraints
`src/simulation.ts`, `src/activity.ts`, and `src/types.ts` still concentrate most runtime and study responsibilities, so each evaluation or mechanism change reopens the same large files. Ecologically, the simulator is still locked to one fungible resource/energy currency, three mutable genome axes, greedy one-step local choice, dominant-only encounter resolution, and single-parent clonal inheritance; those assumptions cap the coexistence structures the project can express. `TaxonHistory.timeline` and per-tick `localityFrames` remain always-on full histories, which keeps long-horizon and counterfactual work expensive before any richer mechanism family is added.
