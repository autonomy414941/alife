# Research Agenda

## Current Direction
Over the next month, tighten the founder-establishment evaluation loop instead of adding another scalar founder-protection knob. Prioritize stricter relabel-null controls that preserve more founder context, prune dead study axes from the active surface, and finish the remaining output and structural cleanup so the next establishment mechanism can be compared against trustworthy artifacts.

## Why This Direction
`newCladeSettlementCrowdingGraceTicks=36` is still the best committed active-clade gain, improving `activeCladeDeltaVsNullMean` from `-36.25` to `-23.75`, but the system still trails the matched null badly. The March 14 ecology-gate horizon follow-up improved the deficit to `-17` at cladogenesis threshold `1.0`, yet it paid a `28.93` to `48.36` persistent-activity regression versus founder grace, while `newCladeEncounterRestraintGraceBoost=2` has already been pruned after a horizon loss. Matched-schedule extraction and founder-age review code now exist, so the next bottleneck is no longer "add one more knob" but "decide whether the remaining gains survive a stricter null and a reliable artifact path."

## Structural Constraints
`src/activity.ts` (2192 lines) still mixes activity analytics, relabel-null study assembly, and comparison logic, while `src/simulation.ts` (2386 lines) still holds most of the runtime loop and accumulates `localityFrames` plus full `TaxonHistory.timeline` histories that grow with horizon. The active study surface is still fragmented across 25 `study:` scripts, 11 of which still bypass the shared atomic JSON emitter. At the evaluation layer, the relabel-null baseline still preserves only clade birth counts by tick, not founder habitat, local crowding, abundance class, or disturbance context, so delta-vs-null gains can still overstate progress.
